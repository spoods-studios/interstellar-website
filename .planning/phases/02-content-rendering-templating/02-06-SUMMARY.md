## Issues Encountered
- Shiki's syntax highlighter tokenizes `[[nodiscard]]` into three separate `<span>` elements (`[[`, `nodiscard`, `]]`, each with its own inline color style) rather than leaving it as one contiguous text run. The plan's acceptance criterion ("shows it verbatim inside a rendered code element") is about the visible text content, not the raw HTML byte sequence, so `tests/technical.smoke.sh` strips all HTML tags (`sed 's/<[^>]*>//g'`) before asserting the literal `[[nodiscard]]` substring is present — this correctly proves the wikilink plugin never touched the code content (T-02-12, structurally guaranteed by Plan 03's text-node-only visitor) while tolerating Shiki's normal per-token markup.
