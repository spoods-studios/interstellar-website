## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| collection join → generated link | Cross-collection lookups produce hrefs; a wrong or unfiltered join publishes something that should not be linked |
| Markdown body → rendered HTML | 55 documents of C++-heavy prose pass through the Markdown pipeline here |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Information Disclosure | Generated cross-link lists (announcement → deep-dives, deep-dive → breadcrumbs) | medium | mitigate | Every cross-collection lookup filters through the shared `isVisible` guard, not only the index queries; an acceptance criterion asserts the drafted-entry case in Plan 08's D-30 check |
| T-02-01 | Tampering | Wikilink-derived `href` in a rendered deep-dive | high | mitigate | Route-level assertion in `tests/technical.smoke.sh` proves the one real wikilink renders as an internal anchor and that no raw bracketed source text survives; the resolver itself cannot emit an external or scheme-bearing URL (Plan 03) |
| T-02-12 | Tampering | C++ attribute misread as a wikilink | high | mitigate | The plugin registers on text nodes only, so code content is structurally unreachable; asserted at route level here and at compile level in Plan 03 |
| T-02-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package installs in this plan |
</threat_model>
