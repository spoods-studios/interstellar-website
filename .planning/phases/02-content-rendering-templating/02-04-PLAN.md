  <action>
Write `src/components/TableOfContents.astro`. It takes the headings array produced by rendering a collection entry, passes it through `buildToc`, and returns nothing when the result is null. Otherwise it renders a labelled navigation block of fragment anchors — one per returned heading, with the depth exposed as a data attribute so `global.css`'s existing indent rule applies. Below the 640px breakpoint the markup is a native `<details>`/`<summary>` disclosure; at and above it the same list renders as the sticky rail. Implement that as one markup tree whose presentation is switched by the stylesheet rather than by two duplicated branches, and add zero client JavaScript — the disclosure widget is the browser's own (D-42, UI-SPEC).

Write `src/layouts/PostLayout.astro` as the single reading layout for all four content trees (UI-SPEC "Layout factoring"). Props: the page `title` (for `<head>`, passed through to `BaseLayout`), an optional `meta` string, an optional `headings` array, an optional `breadcrumbs` array of `{ href, label }`, an optional `prevNext` pair of the same shape, and an optional `backLink`. It renders `BaseLayout`, then in order: breadcrumbs (when supplied), the meta line (when supplied), the TOC component (when headings are supplied), the default slot carrying the rendered Markdown body, and finally the prev/next and back-link block (when supplied).

The body is rendered untouched (D-14): PostLayout emits no title heading of its own, performs no H1 deduplication, and wraps the slot in nothing that adds or removes headings. The Markdown body's own H1 stays the page's primary focal point.

Resolve CONTEXT.md's open discretion item on meta-line placement explicitly: the meta line renders immediately above the slot, as a 14px subordinate label. Placing it below the body H1 would require splitting the rendered body, which D-14 forbids; above the body it still reads as subordinate because the H1 is 30px semibold directly beneath it. Record this choice in the summary.

Use the `set:html` directive for any derived title string rendered into markup, matching the established repo pattern — these titles are git-trusted promoted content and default interpolation would HTML-encode the apostrophes that appear in several real titles.

All styling comes from `global.css`; do not add a `<style>` block that duplicates spacing or colour values already in the stylesheet.
