# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** Phase 1 is an MVP walking-skeleton. Per D-07 (01-CONTEXT.md), the
deployed page is a deliberately **unstyled** ingestion-proof list — throwaway
markup (~20 lines) that exists to demonstrate the ingest → build → deploy
pipeline works end-to-end, not to be a designed page. It is replaced wholesale
by Phase 2 templating (SITE-04 baseline polish, CONT-02 archive rendering).
This contract is intentionally thin: it fixes the small amount of copy Phase 1
does need, and forward-declares the project's default design tokens so Phase 2
inherits a consistent starting point rather than re-deriving one. No custom
CSS file is required or expected in Phase 1.

## Spacing Scale

Declared values (must be multiples of 4) — **forward-declared project default,
not applied to any stylesheet in Phase 1**:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Phase 1's `src/pages/index.astro` ships with no custom spacing
CSS at all — the browser's default `<h1>`/`<ul>`/`<li>` margins apply
unmodified (D-07: unstyled is the explicit intent, not an oversight). This
scale exists so Phase 2's UI-SPEC has a locked starting point rather than
inventing one from scratch.

## Copywriting Contract

Phase 1's only user-facing surface is the single throwaway list page (D-07).
This is the actual copy contract for that page — executor should use these
strings verbatim:

| Element | Copy |
|---------|------|
| `<title>` (browser tab) | `Interstellar Engine — Devblog` |
| Page `<h1>` | `Devblog` |
| List item format | `{title} — {date}` where `{date}` renders as `YYYY-MM-DD` (matches the source filename format, no locale formatting needed this phase) and `{title}` is the collection entry's resolved title (frontmatter `title` if present, else the manifesto's first `# H1` per D-01) |
| Primary CTA | none this phase — no calls to action exist; DIST-03's Discord CTA is Phase 3 scope |
| Empty state heading | not applicable — D-10 requires the build to fail loudly (non-zero exit, naming the offending/missing content) rather than render a page with zero entries; there is no "zero devlog posts" UI state to design in Phase 1 |
| Empty state body | not applicable, see above |
| Error state | not applicable — no custom error page in Phase 1 (SITE-04's custom 404 is Phase 2 scope); a malformed-content build failure surfaces in the GitHub Actions log, not in any rendered page |
| Destructive confirmation | not applicable — no destructive actions exist in this project (static content site, no auth, no forms, no delete flows) |
