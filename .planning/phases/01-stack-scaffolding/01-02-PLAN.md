## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → GitHub Pages | The deploy job holds a short-lived OIDC token (`id-token: write`) authorized to publish to the Pages environment |
| Third-party Actions → CI runner | `withastro/action`, `actions/checkout`, `actions/deploy-pages` execute with the workflow's granted permissions |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | `deploy.yml` `GITHUB_TOKEN` permissions | high | mitigate | Explicit least-privilege `permissions:` block — exactly `contents: read`, `pages: write`, `id-token: write`; never `write-all`. Verify step fails on `write-all`. |
| T-01-SC | Tampering | Unpinned third-party Actions (`@main`) | high | mitigate | Pin every action to its verified major tag (`withastro/action@v6`, `actions/checkout@v7`, `actions/deploy-pages@v5`); verify step fails on any `@main`. |
| T-01-04 | Denial of Service (self-inflicted) | Every-push deploy burns Actions quota | low | accept | D-08 accepts frequent `.planning/`/`vault/` commits each triggering a ~1-2 min run; a path filter's silent-non-deploy risk outweighs the quota cost. |

ASVS L1, block-on-high. Both high threats (T-01-01, T-01-SC) are mitigated in-workflow and enforced by the Task 1 automated verify.
