<objective>
Expand the tracer's `ogImage` prop seam from a constant to a real resolver: the two announcements
that carry a genuine engine plot in their body now embed that plot, and every other page keeps the
default card.

A reader who sees M0.7 or M0.8 shared in Discord sees the actual contrast plot or precession
chart, which is what makes those two links worth clicking. Everything else — the other seven
announcements, all 55 deep-dives, the roadmap tree, the standalone pages and the 404 — keeps the
site card, because inventing imagery for them is forbidden and reusing one milestone's plot
everywhere reads oddly.

Purpose: DIST-02's per-post half (D-48/D-59), the phase's highest-uncertainty mechanism, resolved
empirically in research and now wired.
Output: a pure hero lookup with a loud failure path, its Vite-backed supplier, and the two hero
posts pointing at their own images.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Markdown body image reference → built asset URL | A path written in a content file selects which file the site advertises to embedding clients |

No auth, no session, no user input, no database, no server. Content is git-tracked and
developer-authored via a studio-side promote pipeline.

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-09 | Spoofing | `lookupHero` resolving a content-authored path to an advertised image URL | medium | mitigate | The lookup is basename-keyed against a fixed eager glob of the committed asset directory — it can only ever return a URL for a file already in that directory, and any reference that misses throws. There is no path-traversal surface because no filesystem read is performed with the reference; it is a map key |
| T-03-10 | Tampering | A mistaken promote silently changing which image a post advertises | medium | mitigate | A reference that does not resolve fails the build naming the post and the path (D-48), rather than falling through to the default card where the change would be invisible; a resolving change is a visible content diff under review |
| T-03-11 | Information disclosure | The eager glob emitting every PNG in the asset directory regardless of use | low | accept | The asset directory holds two committed, already-public engine plots; both are published on their own pages today. Nothing private lives there |
</threat_model>
