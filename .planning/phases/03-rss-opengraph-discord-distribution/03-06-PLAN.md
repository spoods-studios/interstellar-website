    Use the exact URLs Task 2 recorded in the SUMMARY. **Discord caches an embed scrape per URL**, so
    if you have already pasted one of these links in this server, append a throwaway query string
    such as `?v=2` to force a fresh scrape — otherwise you will see the pre-deploy result and
    conclude, wrongly, that the tags are broken.

    Paste each of the following into any channel you can delete from afterwards, one at a time:

    1. **An announcement carrying a real hero image** — the M0.7 or M0.8 post. Confirm the embed
       shows: a blue accent bar down the left edge, the small site name above the title, the post
       title, a description sentence, and a **large** image (the engine plot, spanning the embed
       width) rather than a small square thumbnail beside the text.
    2. **A technical deep-dive** — any page under the technical tree. Confirm the same five elements,
       with the default wordmark card as the image. This is the highest-volume link type in
       #technical-devlog, so it is the one that matters most.
    3. **A roadmap milestone detail page.** Confirm the same five elements and the default card.
       Between checks 1 and 3 you have now exercised both image paths.
    4. **The longest deep-dive title you can find.** Confirm that after Discord's own client-side
       truncation the title still reads as a complete thought and that the site name does not appear
       twice.

    Then:

    5. **Feed validation.** Open `https://validator.w3.org/feed/`, paste the deployed feed URL, and
       submit. Require **zero errors**. A self-referencing atom link is emitted deliberately to
       pre-empt the one recommendation the validator predictably raises; if any *warning* remains,
       note its text — a warning is not a failure of this check, an error is.
    6. **The card.** Open the deployed default card image beside the live site header. Confirm the
       wordmark reads in the same typeface and weight as the header wordmark, that nothing is clipped
       at the edges, and that the accent rule is present. If the type resolved to a serif or condensed
       face, the export ran on a machine without the expected fonts and the card should be re-exported.

    If any of the six fails, describe which and what you saw rather than approving — a naked link
    instead of a card on check 1, 2 or 3 almost always means the image URL is relative rather than
    absolute, which is the single most likely silent failure in this phase.
  </how-to-verify>
  <resume-signal>Type "approved" once all six pass, or describe which check failed and what you saw</resume-signal>
