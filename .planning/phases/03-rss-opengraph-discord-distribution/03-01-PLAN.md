<objective>
Prove the phase's distribution spine end to end: a real devblog post URL, when scraped,
produces a complete OpenGraph/Twitter card, and every page on the site routes to Discord.

**Phase goal (ROADMAP, verbatim):** "Shared links produce rich Discord embeds, readers can
subscribe by RSS, and every page routes to Discord — making the Discord-first distribution
strategy actually work."

> **MVP-mode note for the orchestrator:** the ROADMAP Goal line is not in
> `As a … I want to … so that …` user-story form. Per MVP-mode rules the planner does not
> invent one. Recommend running `/gsd mvp-phase 3` if a user-story framing is wanted before
> execution; nothing in this plan depends on it.

This is the phase's **tracer**: one thin, production-quality path through every layer the
phase modifies — config constant → `src/lib/` helper → `BaseLayout` `<head>` and chrome →
`PostLayout` → the announcement route → built `dist/` HTML → committed assertions. It uses
the site-wide default OpenGraph card on every page; Plan 03-03 swaps the two hero posts onto
their own images behind the same `ogImage` prop seam, and Plan 03-02 adds the feed and its
discovery links behind the same footer. Neither is an architectural change to what lands here.

Purpose: catch an architectural dead-end (absolute-URL construction, prop threading through
two layouts, config-load-time loud-fail) after one plan rather than after five.
Output: two new npm dependencies, the invite constant and its build guard, the default OG
card, the description extractor, and a `<head>` block plus Discord CTA on all 86 built pages.
</objective>
