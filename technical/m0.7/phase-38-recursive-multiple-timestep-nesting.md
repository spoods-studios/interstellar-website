## Starting point

Phase 37 shipped a single-level hierarchical-Jacobi (HJS) KDK map: one
Kepler drift per real orbit, one interaction kick, one global step for
every edge in the tree. That step still has to resolve the fastest
orbit in the whole hierarchy вҖ” Io at 1.77 days forces the same `dt`
onto Callisto at 16.7 days and onto the outer Jupiter-Sun orbit at
11.9 years, exactly the coupling flat heliocentric WH had (M0.6 Phase
31: the Moon's О·вүҲ0.40 breaks the single-timestep premise). Phase 38 is
the fix SyMBA (Duncan, Levison & Lee 1998) and Saha & Tremaine (1994)
already worked out for exactly this problem: give each tree edge its
own sub-step count, derived from its own period, and interleave them
so a tight inner pair takes many small steps while the rest of the
tree still advances once per outer `dt`. The fix has to satisfy two
things: the recursive composition itself must be symplectic and
time-reversible, and the per-edge error must scale with that edge's own
interaction strength, not the system's worst mass ratio.

## The sub-step count is derived, not tuned

Each edge declares its own orbital period `P_k` as static tree data
(`HjsOrbitSpec::period`, compiled into `HjsTree::period[]` by
`build_tree` вҖ” Phase-36 precedent: declared, not measured, so the
ladder never depends on live state and stays trivially reversible).
`hjs_nested_ladder()` turns period into sub-step count with one divide,
one multiply, integer ops only:

```cpp
// D-18: n_k = next_pow2(ceil(|dt| * C / P_k)).
const double raw = adt * static_cast<double>(kHjsSubstepsPerPeriod) / p;
int c = static_cast<int>(raw);
if (static_cast<double>(c) < raw) {
    ++c;
}
ladder.n_sub[k] = next_pow2_at_least_one(c);
```

`C = kHjsSubstepsPerPeriod = 20` вҖ” three independent primary sources
converge on it: Beust (2003) В§3.1 calls 1/20 of the shortest period
"a classical choice for MVS-like integrators," Rodet et al. (2019) eq.
7 uses `min_k T_k/20`, and DLL98 give the same number for MVS accuracy.
Rounding up to the next power of two costs compute, not
accuracy вҖ” the Kepler drift is exact at any sub-step size, so a wasted
sub-step is pure overhead. The code documents the bound directly:
"BOUNDED below 2Г— (worst case just under 1.9Г—: e.g. Moon needs 22 вҶ’
runs 32 (~45%), Callisto 12 вҶ’ 16 (~33%))." In exchange, every pair of
cadences in the tree is an exact integer ratio вҖ” no LCM bookkeeping
when levels interleave. `kHjsMaxSubstepLog2 = 20` caps a pathological
`П„/P` ratio at `n вүӨ 2^20`, so a bad declared period fails loudly
instead of silently blowing an array bound or looping to infinity.

## Which pairs kick at which cadence вҖ” the Beust partition

Beust's H_A/H_B split (eqs. 16-17) separates the exact per-orbit Kepler
energy from the interaction Hamiltonian:

```
H_A = ОЈ_k ( p'_kВІ/2m'_k вҲ’ G Ој_k О·_k / r'_k )
H_B = ОЈ_k  G Ој_k О·_k / r'_k  вҲ’  ОЈ_{i<j} G m_i m_j / r_ij
```

H_B decomposes exactly by edge: in a valid HJS tree, every body pair
`{i, j}` sits in opposite subsets of **exactly one** orbit вҖ” its unique
separating edge, computable from the Phase-36 `oloc` matrix with no
floats at all. That gives `H_B = ОЈ_k V_k`, each `V_k` the tidal residual
of edge k (the point-mass center-satellite pull minus the true
group-group pair sum). A leaf edge вҖ” one center, one satellite вҖ” has
`V_k вүЎ 0` identically, so Earth-Moon and Jupiter-Io never contribute a
kick of their own; the tidal wobble they cause shows up in the edges
above them.

The cadence rule assigns each `V_k` to the **fastest** sub-step count
among itself and every edge nested beneath it, not its own edge's rate:

```cpp
for (int k = 2; k <= N; ++k) {
    int fastest = ladder.n_sub[k];
    for (int c = 2; c <= N; ++c) {
        if (c == k) continue;
        if (is_descendant_edge(c, k, c_set, s_set, b_set) &&
            ladder.n_sub[c] > fastest) {
            fastest = ladder.n_sub[c];
        }
    }
    ladder.m_kick[k] = fastest;
```

The reason is physical, not defensive: `V_k`'s leading term is the
quadrupole content generated by whatever is moving fastest inside the
group it spans, so sampling it at its own edge's slower rate aliases
that motion вҖ” the same failure mode Wisdom & Holman (1992) named as
stepsize resonance. Sampling everything at the fastest nested rate
instead means the two-clump Sun/(Earth,Moon)/(Jupiter,Io,Callisto)
config gets exactly two distinct nonzero kick rates: the Sun-Earth/
Sun-Moon edge kicks at the Moon's cadence, and both Jupiter-side edges
kick at Io's вҖ” the multi-rate interleave this recursion exists for.

## The recursion: DLL98's palindrome, not ST94's asymmetric tick

Saha & Tremaine's `TICK` procedure is the textbook per-planet-cadence
recursion, but its reversibility proof leans on a commutation relation
(`{H_Kep,i, H_int,j} = 0` for `j > i`) that holds for their Jacobi-chain
grouping and does **not** hold for an HJS tidal residual вҖ” an outer
edge's `V_k` depends on inner Jacobi coordinates by construction, so
`{H_A,inner, V_outer} вү  0`. DLL98's eq. 16 form sidesteps the whole
question: every level's kick is a symmetric **half**-kick bracketing
the faster levels beneath it, so the composition is a palindrome of
exact flows regardless of what commutes with what:

```cpp
void execute_ladder_level(...) {
    ...
    interaction_kick_masked(tree, mask, xj, vj, half, G, xb_scratch);

    if (level + 1 < ladder.n_levels) {
        for (int r = 0; r < ratio; ++r) {
            execute_ladder_level(tree, ladder, level + 1, ..., trace);
        }
    } else {
        drift_all_orbits(tree, xj, vj, h, G);   // exact Kepler at the deepest level
    }

    // Do not merge this with a neighboring half-kick at repeat boundaries: the
    // algebraic flow would match, but the byte shape and trace palindrome would not.
    interaction_kick_masked(tree, mask, xj, vj, half, G, xb_scratch);
}
```

Drifts only ever happen at the deepest active level вҖ” Kepler flows
commute exactly, so N sub-drifts compose to the same trajectory a
single exact drift would, cost aside. A `TESTING`-only trace ledger
records every kick/drift firing as an integer tick in units of
`П„/n_max`; the test suite asserts the recorded schedule is dyadically
nested (every slower level's firing set is a subset of every faster
level's) and a literal palindrome, without comparing a single float.

The half-kicks are deliberately never merged across a `repeat`
boundary (`K(h/2)В·K(h/2) вҶ’ K(h)` is algebraically identical but changes
the float fold order and erases the explicit palindrome the trace
ledger checks) вҖ” documented the same day (2026-07-09) as "Nested
executor keeps adjacent half-kicks unmerged".

## The degenerate collapse is the regression anchor, not a side test

Every `n_k = 1` вҖ” the whole tree stepping at one rate вҖ” has to
reproduce Phase-37's original `hjs_map_step` exactly, because that *is*
Phase 37 with the wrapper stripped away. Rather than duplicate the
KDK body, `hjs_map_step` was refactored to call `execute_ladder` with a
trivial one-level, all-edges ladder (`single_level_ladder()`), and a
pre-refactor 8-step hexfloat golden was captured before the change
landed, pinning the refactor to a measured bit pattern.
`hjs_nested_step` and `hjs_map_step` now share one executor вҖ” the
all-`n_k=1` bit-identity is structural, not a second implementation
that happens to agree.

## Measuring the error, not assuming it

The per-edge error Hamiltonian (DLL98 eq. 17) predicts `|О”E/E|` scales
with each edge's own interaction strength Оө_k, never the system's
worst global mass ratio вҖ” the property that rules out the flat-WH
failure mode. Two fixtures demonstrate it, both computing Оө and О”E/E
from measurement rather than the analytic order-of-magnitude estimate
research had sketched.

**Eps-sweep.** `sweep_seed(moon_mass_factor)` scales the Moon's mass by
{0.1, 1, 10} while holding declared periods fixed, so the derived
ladder is provably identical across all three runs (a real assertion,
not a tautology). `Оө_edge` is measured in-test from an independent
Beust eq-17 oracle (`V_root(t)` computed directly via `b2g`, separate
from the production kick code), and `max|О”E/E|` from the shared
barycentric energy oracle. The log-log slope across the three points:
**1.028** вҖ” inside the required [0.5, 1.5] band and close to Hernandez
& Dehnen's (2017) вүҲ1 prediction for a WH-family map's leading-order
error.

**Flat-WH contrast.** The same bound-Moon seed that fails M0.6's
`wh_perturbation_eligible` gate at О·вүҲ0.40 runs through both the nested
map and the M0.6 flat democratic-heliocentric `wh_step` (called
directly, test-only, with the production gate asserted still rejecting
it first) at the same coarse outer П„. The original П„ choice
(`P_Moon/20` вүҲ 1.37 d) accidentally gave flat WH enough resolution to
integrate the Moon correctly anyway вҖ” measured contrast only ~4.4Г—,
short of the вүҘ100Г— the requirement needs. Switching to `kTauNested`
(10 d, the same П„ the eps-sweep already used) let the real failure
mode show: at that coarser step, flat WH has no per-edge subdivision
concept at all and takes it directly, while the nested map's declared-
period ladder automatically derives an 8Г— internal sub-cadence for the
Moon edge. Measured result: nested stays bounded at
`max|О”E/E| = 5.637e-9` with the Earth-Moon separation in its expected
band; flat WH degrades **~5591Г—** worse, and the Moon's separation
blows out from 3.84e8 m to ~3.1e11 m вҖ” it escapes to heliocentric
distance. Both ceilings were calibrate-then-locked from these measured
numbers (eps-sweep ceiling 3.0e-6, ~58Г— margin over the worst measured
point; contrast ceiling 3.0e-7, ~53Г— margin), not picked in advance.

## Why the self-term can't be split from the pair sum

DLL98's own multiple-timestep MVS attempt failed for a documented
reason (pp. 10-11): when the all-pairs force and the per-orbit
self-term that compensates it are sampled at different cadences, they
stop cancelling and the integrator drifts. Beust's per-edge partition
keeps each `V_k`'s self-term (`GВ·О·_kВ·Ој_k/r'_k`) in the same mask bit as
the pair forces it compensates вҖ” `interaction_kick_masked` masks
**edges**, never pairs separately from self-terms вҖ” so this failure
mode is structurally unreachable rather than avoided by convention.

## Where it is now (drift since 2026-07-09)

- **A gate finding (2026-07-10):** found
  `interaction_kick_masked` recomputing the barycentric all-pairs
  acceleration once per Jacobi orbit row вҖ” O(NВі), measured 16.5Г—
  redundant work at N=32. Hoisted to a per-body accumulator computed
  once and reused across rows (O(NВІ)), and the worker's warp arm now
  passes its ctor-derived ladder into a new overload instead of
  re-deriving the session-invariant ladder every step. Both fixes are
  bit-identity gated вҖ” same fold order, same skips, byte-identical
  N=9 100-step Jacobi output before and after.
- **Another gate finding (2026-07-10):** comment-only corrections
  across the HJS surface вҖ” the COM-orbit wording, a conditioning bound
  on the interaction-kick subtraction, and the pow2 waste bound now
  documented directly at the cadence-ladder site. No behavior change
  (`.text`/`.rodata` byte-identical per TU).
- **M0.8 Phase 44 (2026-07):** J2 oblateness threads through
  `interaction_kick_masked` as an `OblatenessTable` parameter, added as
  a new overload rather than changing the phase-38 signature вҖ” the
  Newtonian-only call sites are untouched.
- **M0.8 Phase 45:** a per-orbit 1PN correction
  (`PnParams`) was added to the same drift and kick calls, applied
  after the Newtonian and J2 terms in a pinned order, gated on the same
  `edge_selected` cadence mask the Newtonian self-term uses вҖ” the
  DLL98 partition invariant (never split a compensating term across
  cadences) extends to the relativistic term by the same construction.
- As of 2026-07-21, the recursive ladder derivation
  (`hjs_nested_ladder`) and the palindromic executor
  (`execute_ladder_level`) are structurally unchanged since this phase
  вҖ” every call now carries optional, defaulted `OblatenessTable`/
  `PnParams` arguments, but the half-kick bracketing, the no-merge
  policy, and the all-`n_k=1` collapse to `hjs_map_step` are the same
  code this phase shipped.
