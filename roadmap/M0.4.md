## Phase 18: N-Body Snapshot Redesign — ✅ 2026-06-08

**Goal:** replace the single-body `Snapshot` POD + per-publish
full-state-copy seqlock with a fixed-capacity SoA triple-buffer that
publishes an atomic `{index, generation}` and is read in place by render —
O(1) publish, N-capable container while the sim still runs one body.
Closes the M0.3 named blocker (the N-body snapshot redesign) before any
later feature binds to the handoff. Architecture carried from the
2026-06-05 cross-AI decision; 10 phase decisions were locked.

**What it delivered:**
- `Page { vector<Vec3f64> pos, vel; FrameMeta meta }` × 3 in
  `worker_thread.hpp` — each page reserved to `Config::body_capacity` once at
  construction, never grown on the hot path; active `count` ≠ capacity,
  readers iterate `[0,count)`; single shared immutable body-id table
  built once, not per-page — slot `i` ↔ same body across all pages
- O(1) publish: one packed 64-bit `{generation << 2 | page-index}` atomic
  (`pack`/`gen_of`/`idx_of`, public constexpr) stored with the M0.2 seqlock
  fence pattern — odd marker → release fence → in-place back-page fill →
  release fence → even commit → 3-page reclaim rotation (never 2 — the P14
  TSan kill); no per-publish memcpy, the page fill is the step loop's write
- Read-in-place `latest_snapshot()` returning `SnapshotView` (owns its copied
  bytes; `state_of(i)`/`pos_at(i)`/`vel_at(i)` per-slot reconstruction);
  torn `count` clamped before the generation recheck discards the frame;
  documented as lock-free, wait-free only under a bounded publish rate
- `FrameMeta` per page — `sim_time`, `origin`, whole-system
  `rel_energy_error` (the close-encounter gate quantity), `count` —
  swapping atomically with the index; `Snapshot::period_estimate` deleted
  (no N-body period; `KeplerElements::period` untouched), HUD reduced to
  `T+` + `ΔE/E`
- A contention soak: 4-slot coherence-stamped frames (every component
  of every slot = the same strictly increasing stamp — the N-body lift of
  M0.2's `x==y==z` trick), `torn == 0` + monotonic over ≥5 s / 60 s opt-in;
  double-publish-window case asserting `jumps_ge_2 > 0` so the P14 failure
  window is provably exercised; `INTERSTELLAR_TESTING`-only
  `publish_stamped_frame_for_test` writer hook reproducing the production
  protocol byte-for-byte; scoped TSan suppression + `_tsan` ctest variants
- Exit gate: 163/163 ctest green Debug + Release + TSan (5× parallel runs),
  Catch2 152 → 161 cases, no M0.2/M0.3 math-lock file touched, N=1
  trajectories bit-identical

**Key decisions:** triple buffer + read-in-place seqlock reader over
reader-swaps triple buffering (reader stays a pure reader — no RMW;
extends the TSan-proven M0.2 seqlock pattern to N bodies) and over
per-body seqlocks/hazard pointers (both rejected during the
architecture review); construction-time heap capacity over
`constexpr MaxBodies = 256` (a craft tier raises the cap with no
recompile); parallel `pos[]`/`vel[]` over x/y/z scalar-split SoA
(premature SIMD on a render-facing snapshot); one packed word over
separate index/generation atomics; `Vec3f64` reuse over a
snapshot-local POD; whole-system energy error over per-body (it is
the close-encounter gate quantity).

**Deviations:** three in the first plan — three unenumerated
`latest_snapshot()` consumers migrated with no tolerance change;
`ctest -R snapshot` matched nothing until a `snapshot` token was
injected into TEST_CASE titles; dead includes dropped after the
period removal. Two in the second plan, both surfaced only by
parallel TSan runs — the coherence check was made count-agnostic
after the production count=1 seed frame false-positived as a tear; an
absolute-page precondition was de-flaked. Post-phase: Phase 21
promoted `state_` to a `states_` vector and enabled real N>1 through
the unchanged container; a later fix extracted `run_due_steps()`/
`publish_with()` and added the zero-alloc out-param reader;
`FrameMeta` has since grown telemetry fields (regime, AMD,
`warp_active`, per-clump drift) — the packed-word publish protocol is
unchanged as of 2026-07-21.

## Phase 19: N-Body Direct Force + State Promotion — ✅ 2026-06-08

**Goal:** evaluate gravity for N gravitating bodies exactly and
deterministically — direct O(N²) Newtonian in float64, symmetric `i<j`
accumulation, fixed summation order — and run the existing Leapfrog/Yoshida4
kernels across an N-element state vector, with N=1 reproducing M0.3
bit-for-bit. 8 decisions were locked, one reframed same-day by a
research finding.

**What it delivered:**
- `engine/src/nbody_force.cpp` / `nbody_force.hpp` — `compute_accelerations`: central-μ pre-fill through the verbatim M0.3 `evaluate_force(mu, r)` expression, then the REBOUND BASIC symmetric `i<j` accelerate-direct loop (`dr = r[j]−r[i]`, `inv_r3 = 1/(r2·√r2)`, `prefact = G·inv_r3`, `accel[i] += prefact·m_j·dr`, `accel[j] −= prefact·m_i·dr`). No tree/FMM, no softening, no superaccumulator — plain `+=` under `-ffp-contract=off` + `FE_TONEAREST`. Declared the locked reference summation order
- `BodyProps { id, mass, mu, gravitating }` + `BodyTable` span view extending Phase 18's shared immutable table; the span is the canonical body order. Accumulate-acceleration makes the massless tier branch-free — a mass=0 body exerts nothing and still feels everything, no `1/0`
- Integrator seam swap: per-body `bound_force_fn` function pointer retired for a whole-vector `bound_accel_fn` `std::function` fired at both KDK kick boundaries (pre-drift a0, post-drift a1); worker dispatches `span<State>{&state_, active}` through a worker-owned `accel_` scratch sized at construction — no hot-path allocation
- Whole-system energy diagnostic `Σ½vᵢ² − Σ μ_central/rᵢ − Σ_{i<j} G·mᵢmⱼ/rᵢⱼ` reducing exactly to M0.3's `½v² − μ/r` at N=1; N-aware NaN/Inf filter naming the offending body slot
- Named N=1 bit-identity gate (`test_nbody_n1_identity.cpp`, `==` not `Approx`, 6 radii LEO→lunar) + full math-lock suite unchanged — zero M0.3 numbers moved
- A same-binary A/B determinism gate: same N=2 seed twice in one binary, bit-identical r/v over ~1000 Yoshida4 substeps at the pure-mutual path (`mu_central = 0`), plus permuted-input order-stability; Release + Debug + TSan each 170/170 with an identical 71-entry ctest surface

**Key decisions:** direct O(N²) exact force — tree/FMM θ-error injects
artificial periastron precession and breaks the separable
Hamiltonian; no Plummer softening — close approaches belong to the
integrator (Phase 20); same-binary determinism now, cross-platform
tolerance deferred to Phase 21; the central-μ source was retained so
N=1 stays a test particle in a fixed μ-field (a same-day research
reframe — a literal self-less body coasts in a straight line and
fails every math-lock); accumulate acceleration, not
force-then-divide; the `std::function` seam is built once at the
ctor.

**Deviations:** the mutual loop's gravitating gate shipped gating
both pair sides, zeroing force a non-gravitating body must feel —
caught by the first test run, fixed in-phase to an outer-`i`
exerts-side gate; the seam swap broke seven Config-consuming test
files beyond the plan's list, repaired via the `force_override` seam.
Post-phase: the M0.4 gate repinned masses as GM/G by construction and
replaced the outer-`i` gate with per-side gates after a slot-order bug
left a non-gravitating body at slot 0 force-free; M0.5 Phase 26 split
the kernel into active/test-particle blocks and locked the
determinism contract; a later fix added `GravConstant`/`CentralMu`
strong types. The file is the engine's locked deterministic force
kernel today.

## Phase 20: Close-Encounter Integration — ✅ 2026-06-08

**Goal:** stop the fixed-step energy bleed through close approaches
without shipping a known failure mode — closing the deferred
close-encounter blocker: mutual Hill-radius detection with hysteresis
and a whole-system adaptive IAS15-class Gauss-Radau fallback during
encounter windows; energy-error a test assertion, never the trigger.
Locked by the 2026-06-05 cross-AI decision plus the 2026-06-08
"IAS15 clean-room" decision entry.

**What it delivered:**
- `ias15_integrator.hpp/.cpp` — clean-room IAS15 from Rein & Spiegel 2015 (arXiv:1409.4779; REBOUND/GRSS are GPLv3, incompatible with the planned Apache-2.0/MIT license — no GPL source opened): 8 Gauss-Radau nodes + g↔b matrices pinned to 25 sig figs, eq.(5) predictor-corrector recurrence, Kahan-compensated updates, `(ε_b/b̃₆)^(1/7)` step controller with a hand-rolled 7th-root on `+,−,×,÷` only (bit-exponent seed + 20 fixed Newton iterations, 0–3.6e-16 vs `std::pow`)
- `integrate_window` — accept/reject adaptive loop landing exactly on the `step_dt` boundary so the worker accumulator/clock/publish cadence stay untouched; worker-owned `Ias15Scratch` sized to capacity, no hot-path alloc
- A standalone validation gate before any wiring: Kepler e=0.3 position 7.98e-13 over 50 periods; 15th-order convergence anchor 2^13.93 in the a-priori [2^13, 2^17] band; e=0.9 |ΔE/E| 2.88e-14 vs Yoshida4's 1.49e-4 (5.17e9× gap); same-binary bit-determinism
- `encounter_detector.hpp/.cpp` — `hill_radius` in the MERCURIUS changeover form (`R_Hill = a·((m_i+m_j)/3M)^(1/3)`, a = mean heliocentric distance — mutual-separation `a` proved degenerate) + `detect_regime` hysteresis pure function (`k_in=3.0`/`k_out=3.6` config-exposed, canonical id order, no energy/wall-clock/unordered reduction); deep-plunge straddle CONTRAST at pinned 60 s step_dt: bare-Hill k=1 MISSES, conservative k_in=3.0 CATCHES
- Regime-switch seam in BOTH worker step loops — IAS15 is NOT a third `IntegratorMethod`; worker-driven no-encounter bit-identity vs a forced-REGULAR `k=0` baseline, `==` over 80 published frames
- A bounded-energy regression: self-consistent N=3 impact-parameter flyby through the seam, max |ΔE/E| 4.58e-15 (straddle 8.27e-15), locked at the ε_b-aligned 1e-9 with the ~5-order Brouwer-√n margin documented; bounded-not-monotone via decreasing fraction 0.33
- Phase gate: 207/207 green Debug + Release + TSan; M0.3 10⁵-step energy gate bit-identical (`test_energy_conservation.cpp` unmodified)

**Key decisions:** a whole-system adaptive fallback over
hard-stop/per-pair hybrid (MERCURIUS-style hybrid deferred to M0.5 as
a non-rewrite reuse); geometric trigger, energy strictly a test
assertion — symptom-as-trigger is reactive and circular; clean-room
from the paper for license safety; standalone validation before
wiring; calibrate-then-lock for k and the energy bound;
conservative-k deep-plunge mitigation in-phase, only a
periapsis-prediction trigger deferred.

**Deviations:** four auto-fixed transcription/logic bugs, all caught
by the wave's own tests same-day — the d-matrix as the true c⁻¹
(round-trip 2.0039→2.0), a b-series double-integration exponent, the
accept/reject loop force-accepting the oversized first full-window
trial, and the mutual-separation Hill `a` degeneracy; the
bounded-energy scenario became an impact-parameter flyby (a bound
high-e infall needs ~7.7M steps, intractable for a unit test).
Post-phase: a bound-satellite-pair exemption, a zero-mass pair gate,
and (in M0.5) a predictive trigger plus WH↔IAS15 handoff.

backlog item captured 2026-06-04 during Phase 15.5's visual UAT.

**Delivered:**
- `orbit_demo.cpp` emits a machine-parseable `[telemetry]` line to stderr
  every `kTelemetryFrameCadence = 120` rendered frames (~1 line / 2 s at
  60 fps): sim-clock `t=`, whole-system `dE/E=` relative energy drift, and

  Earth orbit is watchable in ~30 s wall-clock (was ~8.5 h); `time_scale_min`
  unchanged; kept an exact power-of-two (preserves the long-horizon
  exact-double accumulator reasoning) and bounds worst-case
  steps/iteration to `scale·wall_dt_clamp/step_dt = 2²⁰·0.25/300 ≈ 873`,
  inside the Fiedler spiral-of-death guard
  (`wall_dt_clamp_seconds = 0.25`, unchanged)
- An invariant (`time_scale` never mutates `step_dt`) asserted by a new
  test driving `tick()` at scale 1.0 and at `time_scale_max`, checking
  `sim_time` lands on exact `step_dt` multiples in both regimes; the
  existing clamp-bounds test updated to assert against the named
  `time_scale_max`/`time_scale_min` constants instead of literals
- Full 3-config gate: Release 216/216, Debug 216/216 (+1 net-new,
  additive), TSan contention/liveness suite green with existing
  suppression config; the deterministic force kernel untouched
  (`nbody_force.cpp` not in this phase's diff); no
  M0.2/M0.3/Phase 19/Phase 20 math-lock tolerance moved

**Key decisions:** visualization/HUD ruled out of scope for the
engine, belonging to the game layer — the engine owes physics
correctness plus at most a legible debug overlay, not game-quality
visualization; stderr telemetry chosen over an on-screen ΔE/E
envelope because logs are parseable, deterministic, and don't depend
on a fragile in-engine text renderer (2026-06-09 decision). The

**Deviations:** none — single plan, single commit, same-day close.

ahead of it since 21.7's 100-yr stability run needed the raised
