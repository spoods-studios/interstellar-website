## Phase 36: HJS Coordinate Core — ✅ 2026-07-01

**Goal:** build the hierarchical-Jacobi (HJS) hierarchy-tree representation
and the barycentric↔Jacobi coordinate transforms every later M0.7 phase
runs on — correct iff coordinates round-trip bit-clean and the kinetic
energy diagonalizes; no map, no Kepler drift, no handoff here.

**What it delivered:**
- `HjsOrbitSpec` (declarative per-orbit centers/satellites) compiled by
  `build_tree` into the `oloc(orbit k, body i) ∈ {+1,−1,0}` dense matrix,
  never a hand-written `oloc`; a dense N×N matrix over an O(N) tree-walk,
  the bit-exact-against-reference choice at seed scale
- Forward matrix `mat` per Beust eq. 13; closed-form inverse `umat`
  (`vsat=η/(μ+η)`, `vcen=−μ/(μ+η)`) rather than numerical inversion, so
  `g2b(b2g(x))==x` bit-clean
- `b2g`/`g2b` apply the SAME matrix to position and velocity — the paper's
  `(M⁻¹)ᵀ` is for conjugate momenta, a different vector; building a
  transposed-inverse velocity matrix is the canonical HJS porting bug.
  `g2b` folds `k=1..N` (not the reference implementation's `k=2..N`), a
  general inverse for arbitrary input frames rather than only the COM frame
- Every mass-weighted accumulation is an ascending left-fold, replicating
  the reference implementation's summation order for cross-language bit-
  identity
- `valid()` — a purely combinatorial foreign/inner/outer tree-consistency
  predicate over body-set bitmasks, no floats; massless test particles need
  no hot-path branch — the `!=0.0` sparsity skip handles a test particle's
  zero-mass row naturally
- A property suite (6 test cases): round-trip within a derived `C=4N` ULP
  bound over 10k random valid trees; kinetic-energy diagonalization
  `K=M·diag(1/m)·Mᵀ`; `valid()` iff K-diagonal; a 3-body crossing-chain
  negative control; a test-particle fixture; and a committed cross-check
  golden pin (landed as a self-consistency capture, a real Fortran
  cross-check named as a pending documented recipe)
- 453/453 tests pass at phase close; all M0.2–M0.6 locks green;
  determinism clean (`nbody_force.cpp` untouched — HJS is a new, separate
  translation unit)

**Key decisions:** hierarchical-Jacobi coordinates as a distinct module
alongside the existing flat democratic-heliocentric `wisdom_holman.hpp`,
not a replacement (M0.7 doesn't remove the DH map); fixed-capacity dense
worker-owned storage (`kHjsMaxBodies=32`), zero heap allocation, mirroring
an earlier M0.6 precedent; runtime re-hierarchization (accel-ratio capture/
escape) named an explicit M1.x deferral — Phase 36 ships a static declared
tree only.

**Deviations:** none against the plan. Post-phase: the real Fortran
cross-check promised earlier landed at the review pass (2026-07-10) rather
than in-phase — measured 0 ULP deviation on every real Jacobi orbit; a
gate-found entry-bounds gap in `build_tree` and an unenforced IEC-559
assumption were fixed the same day.

## Phase 37: Nested WH Map — ✅ 2026-07-01

**Goal:** split the HJS Hamiltonian per Beust (2003) eqs. 16–17 and run a
2nd-order KDK map on top of the Phase-36 hierarchical-Jacobi coordinate
core — exact per-orbit Kepler drift plus a small interaction kick, so a
bound satellite (the Moon, which flat heliocentric WH mis-models) integrates
correctly at warp.

**What it delivered:**
- `hjs_map_step` — a KDK composition (`kick(τ/2)·drift-all(τ)·kick(τ/2)`)
  over the Phase-36 tree: per-orbit exact `kepler_step` drift at
  `G·(η_k+μ_k)` for each real orbit, the COM orbit as free translation
  only; a fresh eq-17 kick kernel (all-pairs barycentric minus the
  per-orbit self-term) — `nbody_force.cpp` neither called nor modified
- Two-body collapse structural probe: an all-massive N=2 tree's kick is
  identically zero, so the step reduces to a bare `kepler_step(dt)` —
  catches a wrong Hamiltonian split before any energy hunt
- Sun/Earth/Moon bound-Moon seed, the inverse of the M0.6 η-gate
  rejection — over 3 Moon periods (600 steps) the Moon stays bound
  (separation band 0.75–1.25× the seed radius) while the EMB sweeps more
  than 10% of its 1 AU heliocentric arc
- Barycentric energy held bounded: max `|ΔE/E| = 6.7e-11` against a 1e-4
  demonstration ceiling, near-zero LSQ slope; time-reversibility within a
  derived `N·1e-10` tolerance (the M0.5 WH-determinism reversal floor),
  not bit-exact — the fixed-iteration Kepler corrector isn't
  bit-reversible
- 7 review-gate findings fixed in-phase: the COM row silently frozen
  instead of free-translating under a boosted frame; degenerate massless
  orbits Kepler-drifted on a bogus Jacobi row; missing scratch-precondition
  asserts; a massless test-particle's barycentric state lost across the
  `b2g`/`g2b` round-trip; two independent oversized-N OOB paths
  (`hjs_map_step` and `valid()`); a coincident/non-finite separation
  silently swallowed instead of propagating; landed as an explicit
  `hjs_map_eligible` predicate (mirrors the WH eligibility gate) that
  refuses a massless test-particle tree outright
- Determinism diff empty, libm-free (confirmed only `kepler_step`/`b2g`/
  `g2b`/`det_sqrt` as undefined symbols), 467/467 Release ctest green

**Key decisions:** standalone `hjs_map` translation unit, no live worker
wiring this phase (Phase 39's job, mirroring the Phase-36 precedent); a
fresh self-contained kick kernel rather than reusing `nbody_force.cpp`,
since H_B subtracts the self-term the drift already carries and reuse
would double-count it; plain KDK, no symplectic corrector at this layer
(the corrector is the Phase-39 handoff operator); loose demonstration
energy bound plus derived reversibility tolerance rather than the formal
Brouwer-√t suite (deferred to Phase 40's math-lock); the massless
test-particle contract as an explicit, queryable refusal rather than a
silent no-op (promoted the same day to a documented decision naming the
Phase-36 test-particle-coordinate tier as the concrete blocker).

**Deviations:** the plan specified a massless-TP tier that drifts under
its center's `G·M_k` while sourcing no kick; the landed behavior after the
gate-fix round is outright refusal via `hjs_map_eligible` — a queryable
ineligibility, not the planned collapse, because the Phase-36 transform
has no faithful Jacobi row for a massless body to drift on. Two additional
correctness bugs (COM row frozen, degenerate orbits drifted anyway) and
three hardening gaps (missing asserts, unchecked oversized-N, swallowed
non-finite separations) surfaced and were fixed during the same-phase
gate-fix round rather than deferred.

## Phase 38: Recursive Multiple-Timestep Nesting — ✅ 2026-07-09

**Goal:** add SyMBA-style recursive multiple-timestep sub-stepping on top
of the Phase-37 single-level HJS KDK map — tightly-bound inner subsystems
(Moon ~27 d, Io ~1.77 d) take recursively subdivided sub-steps interleaved
inside the outer drift-kick-drift, while the overall composition stays
symplectic and time-reversible, and the energy error tracks per-edge
interaction strength rather than the global mass ratio that sank flat WH.

**Delivered:**
- Declared per-edge orbital period (`HjsOrbitSpec::period` →
  `HjsTree::period[]`) and a `sep[i][j]` unique-separating-orbit table,
  both compiled by `build_tree` from combinatorics alone — no floats in
  the partition
- `hjs_nested_ladder()` — libm-free `n_k = next_pow2(ceil(|dt|·C/P_k))`
  per edge (C=20, three independent primary citations), fastest-descendant
  kick cadence `m_k`, and a dyadic level ladder; a bad declared period or
  over-cap τ/P ratio is refused loudly, never silently clamped
- `interaction_kick_masked()` — the Beust eq-17 H_B kick restricted to a
  partition mask (per Duncan/Levison/Lee 1998), keeping every self-term in
  the same edge as the pair forces it compensates (the multiple-timestep
  failure mode this structurally rules out)
- A shared recursive palindromic ladder executor: half-kick open →
  nested faster level or exact `kepler_step` drift at the deepest level →
  half-kick close, never merging adjacent half-kicks across a repeat
  boundary; `hjs_map_step` now runs through the same executor with a
  trivial one-level ladder, pinned bit-identical to Phase 37 by an 8-step
  hexfloat golden captured before the refactor
- `hjs_nested_step()`/`hjs_nested_eligible()` public API, plus a
  testing-only trace ledger proving dyadic alignment and palindrome
  ordering with pure integer ticks
- An eps-sweep measured `|ΔE/E|` vs measured per-edge ε across Moon-mass
  ×{0.1, 1, 10} — log-log slope **1.028** (required [0.5, 1.5])
- A flat-WH contrast: the bound-Moon seed that fails M0.6's
  perturbation-ratio gate (η≈0.40) stays bounded under the nested map
  (`max|ΔE/E| = 5.637e-9`) while the M0.6 flat `wh_step`, run directly at
  the same τ, degrades **~5591×** and the Moon's separation blows out to
  ~3.1e11 m (heliocentric distance) — the production eligibility gate
  asserted still rejecting the config throughout
- Determinism (`nbody_force.cpp`)/`kepler_universal.hpp` empty diff across
  every phase commit; Release 484/484, Debug 480/480 at phase exit

**Key decisions:** `n_k` derives from a static declared period, never live
orbital elements, so the ladder is deterministic and trivially reversible;
rounding up to power-of-two (a Saha & Tremaine 1994 practice satisfying
SyMBA's integer-ratio requirement) so every pair of cadences shares an
exact integer ratio; kick cadence assigned by the fastest nested
descendant, not an edge's own rate, to resolve the fastest angle each
interaction term depends on; validation on Sun/Earth-Moon plus a minimal
Jupiter clump (Io/Callisto, ~9.4× period spread) to exercise true per-orbit
multi-rate recursion and a two-clump interleave; two loose-bound
demonstrations accepted for the energy-tracking requirement this phase,
deferring the formal Brouwer/Jacobi-vs-DHC property suites to Phase 40.

**Deviations:** the flat-WH contrast fixture's τ was changed from the
plan's literal `P_Moon/20` to the already-established `kTauNested` (10 d,
already used by the eps-sweep) after a standalone probe showed the plan's
finer τ accidentally gave flat WH enough resolution to integrate the Moon
correctly, understating the contrast at ~4.4× instead of the required
≥100×; the coarser, already-established τ produces the genuine ~5591×
failure the requirement is meant to demonstrate — no seed or assertion
shape changed, only the one constant.

**Goal:** wire the nested HJS map live into the `PhysicsWorker` as the warp

keep-unsynchronized peek for display — consuming M0.6's "validated seat,

**What it delivered:**

  session steps on, extracted bit-identically from the nested map's
  `b2g`→ladder→`g2b` interior; `hjs_map_kick` promoted to production ABI
  as the corrector's B operator
- A Chambers (1999) / Wisdom (2006, eq. 35) third-order two-stage KDK
  corrector pair, coefficients pinned to ≤1 ULP, measured 484× boundary-
  energy improvement on the single-level map — and measured ~31×
  *degradation* on an active nested ladder (genuine physics: the corrector
  re-injects the single-level offset Phase-38 substepping already
  cancelled), leading to a same-day amendment: apply the corrector only
  when the derived cadence ladder is trivial (all n_k=1); an active
  nested ladder enters/exports corrector-free via bare `b2g`/`g2b`
- `hjs_warp_enter`/`hjs_warp_export` — one b2g + corrector on entry;
  copy → invert-corrector → g2b on exit, live session buffers passed

  bug is unrepresentable, not merely tested against
- `Config::hierarchy` (declarative HJS routing — invalid/ineligible
  declared trees throw `std::invalid_argument`, never a silent flat
  fallback) + `Config::warp_in_scale{2048}`/`warp_out_scale{512}` (a
  two-octave `time_scale` hysteresis band, safe specifically because
  `time_scale` is exogenous input, not physical state)
- A keep-unsynchronized peek at the existing snapshot publish point —
  `FrameMeta::warp_active`/`SnapshotView::warp_active` observable;
  peek-heavy and peek-free runs publish bit-identical (`==`) final states
- Composed-system acceptance: warp→flown→warp round trip (`|ΔE/E|` 8.60e-15
  vs a 3.2e-13 derived bound), a 6-cycle band-thrash (12 transitions
  exactly, 0 in-band), a boosted-frame (v_com≠0) COM free-drift fixture
  (position residual 3.05e-5 m vs a frozen-COM miss ~1e5 m), and a queued
  mid-warp maneuver matching a standalone export→mutate→re-enter reference
  bit-for-bit
- Determinism preserved (`nbody_force.cpp` diff empty); `hjs_handoff.cpp`
  libm-free (comment-stripped grep gate); Release 537/537, Debug 533/533
  fresh full-suite at phase close

**Key decisions:** declarative seed-driven routing (no runtime heuristics);
the corrector pinned to Wisdom-2006 eq. 35 / Chambers 1999 KDK
coefficients, then amended same-day to gate on ladder triviality once the
nested-map degradation was measured; the hysteresis band width derived
